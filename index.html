The simple model

Your repository holds the files that Pages can publish.

Antigravity edits

Your cloned repo opens in Antigravity for changes and source control.

3

Pages publishes

GitHub Pages turns the repo files into a live website link.

Common question

Why not just use the files already in Antigravity?

You can edit local files in Antigravity, but GitHub Pages cannot publish a normal folder on your computer. Pages publishes from files stored in a GitHub repository.

For this class, upload the finished project to GitHub first. Then clone that repo in Antigravity so your editor is working with a folder that already knows where to push changes.

Antigravity + GitHub

Can Antigravity push new files?

Yes, once the folder is connected to a GitHub repository. New files still need to be saved, committed, and pushed. Antigravity does not automatically publish every file the moment you create it.

Local setup

Does Git need to be installed?

Yes for local source control. You do not need to type Git commands for this tutorial, but Antigravity needs Git available on the computer to clone, commit, and push.

Recommended student path

  1. Check that index.html is at the top level of your project folder.
  2. Create a new public GitHub repository.
  3. Upload the contents of your project folder.
  4. Turn on GitHub Pages from main and / (root).
  5. Clone the repo in Antigravity.
  6. Edit one line, commit, and push.
  7. Submit your GitHub repo URL and live Pages URL.
