# Instructor Notes

## Recommended Classroom Sequence
1. Show the simple model: GitHub stores the site. Antigravity edits the site. GitHub Pages publishes the site.
2. Explain why the class starts in GitHub: Pages publishes from GitHub, not from a normal local folder.
3. Have students check that `index.html` is at the top level of their project folder.
4. Have students create a new public GitHub repo.
5. Have students upload the contents of the project folder.
6. Have students turn on GitHub Pages from `main` and `/ (root)`.
7. Have students copy the live Pages URL.
8. Have students clone the repo in Antigravity.
9. Have students make one small edit, commit, and push.
10. Have students refresh the live site and submit both links.

## Demo Live
- The difference between uploading a folder and uploading the files inside the folder.
- Where `index.html` should appear on the first page of the repo.
- Settings -> Pages -> Deploy from a branch -> `main` -> `/ (root)`.
- The green **Code** button and the repo URL.
- Antigravity clone, edit, commit, and push.
- What happens if students open a normal folder in Antigravity instead of cloning the repo.
- How long GitHub Pages can take to update.

## Common Misunderstandings
- Students upload one extra folder layer.
- Students think saving in Antigravity updates GitHub.
- Students think opening a folder in Antigravity automatically connects it to GitHub.
- Students commit but forget to push.
- Students do not have Git installed, so clone or Source Control does not work.
- Students use `Index.html` instead of `index.html`.
- Students break image or CSS paths because capitalization or folders changed.
- Students expect the Pages URL to update instantly.

## Suggested Language
GitHub stores the site. Antigravity edits the site. GitHub Pages publishes the site.

Antigravity can edit any folder, but GitHub Pages can only publish files that are in GitHub.

Commit saves the change. Push sends it back to GitHub.

## Suggested Submission Requirement
Students submit:

- GitHub repository URL
- live GitHub Pages URL
